# 브리프 Q2-20260610-memo-mode — 간단 메모 입력 모드 (50자 제한 완화 + 프롬프트 분기)

## 배경 (Why)
AUDIT-2026-06-10 §3 가설②: 수기 폼은 무겁고(7섹션 ~26필드), 50자 미만 메모는 AI 보조를 아예 못 받는다(서버 ~156 하드 거부, 클라 ~6653 동일). 프롬프트는 8~15문장 내러티브를 강제해 짧은 입력에 부적합. "짧게 쓰고 AI가 구조화해주는" 중간 지대를 만든다.

## 목표
20자 이상의 짧은 메모도 추출 파이프라인을 태울 수 있게 하되, 메모 모드에선 (a) 내러티브를 2~4문장으로 축소 (b) 환각 금지 강화 — 근거 없는 필드는 빈 값 (c) confidence 보수적.

## 스펙
### 1. API (`api/extract-session.js`)
- 입력 계약 확장: body 에 `inputMode?: 'memo' | 'transcript'` (기본 'transcript').
- 길이 검증 분기: `transcript` 모드 ≥50자(현행), `memo` 모드 ≥20자. 둘 다 아니면 400 (에러 메시지에 두 기준 명시).
- `memo` 모드에서 audio 는 거부하지 않아도 됨(무관) — transcript 텍스트만 대상으로.
- `buildSystemPrompt(isMemo)` 파라미터화:
  - memo=true 일 때 PASS 1(내러티브) 지시를 교체: "2-4문장, 메모에 명시된 사실만, 추측·보강 금지".
  - VALUE RULES 앞에 추가 지시: "입력이 짧은 메모이므로 대부분의 필드는 빈 값이 정상이다. 메모에 직접 근거가 없는 필드는 반드시 빈 값으로 두고, 채운 필드도 confidence ≤ 0.7 로 보수적으로."
  - 22필드 정의 자체는 변경 금지 (STRUCTURED_FIELD_KEYS 불변).
- `buildUserPrompt` 헤더에 `- input type: short memo (not a full transcript)` 1행 추가 (memo 시).
- EXTRACTION_VERSION 상수가 있으면 `'2026-06-10.2'` 로 올려라 (Q3 선적용됨 — 프롬프트 의미 변경이므로).

### 2. 프론트 (`public/index.html`)
- STT 패널 textarea 안내문/placeholder 갱신: "녹취 원문(50자↑) 또는 간단 메모(20자↑) — 메모는 핵심만 적어도 AI가 구조화합니다."
- `runSttExtract`(~6646): 텍스트 길이 20~49자면 `inputMode:'memo'` 로 전송하고 상태 메시지 "메모 모드로 분석 중…" 표시. 50자 이상이어도 **명시 토글은 만들지 않는다** (자동 판정, UI 단순 유지). 20자 미만은 현행대로 거부 (문구만 "최소 20자" 로).
- 메모 모드 응답 처리 후 안내: "메모 기반 초안 — 빈 필드는 직접 채워주세요" (기존 setSttStatus 패턴).

## CAN touch
- `api/extract-session.js` · `public/index.html` (STT 패널·runSttExtract 주변만).

## MUST NOT
- STRUCTURED_FIELD_KEYS·정규화 로직·SSE 스트림 형식 변경 금지. transcript(50자↑) 경로의 기존 동작 회귀 금지. RLS·DB 금지. git 금지.

## 추가 지시 (Addendum 2026-06-10, 메인 검증 중 발견)
라이브 프로덕션에서 `applyI18n()` 실행 시 `.stt-subtitle`·`#sttTranscript`·`#fNarrative` 가 사전에 없는 키 때문에 **리터럴 "stt_subtitle"/"stt_placeholder"/"narrative_ph" 로 표시되는 기존 버그 실측 확인** (브라우저 DOM 검사, lang=ko). Q2 의 새 카피도 이대로면 덮인다.
→ `i18n` 사전(~3798)의 **ko·en·ja 세 섹션 모두**에 `stt_subtitle`·`stt_placeholder`·`narrative_ph` 3키를 추가하라. ko 는 HTML 의 새 카피와 동일 문구, en/ja 는 자연스러운 번역 (메모 20자↑·녹취 50자↑ 기준 포함). 다른 키·구조 변경 금지. [완료 2026-06-10]

### Addendum 2단계 — 라이브 DOM 전수 검사 결과 (메인 실측)
배포 폼 뷰에서 리터럴 키 노출 12건 확인. 위 3키 외 **나머지 9키도 동일하게 ko·en·ja 사전에 추가**:
`stt_title` · `stt_extract` · `stt_hint` · `stt_save_raw` · `narrative_title` · `narrative_sub` · `checkin_date` · `checkin_channel` · `checkin_memo`
ko 값 = 해당 HTML 요소의 기존 한국어 기본 텍스트 그대로 (data-i18n 속성이 붙은 요소 내용), en/ja 는 자연 번역. stt_hint 등 길이 기준 언급이 있으면 20자/50자 기준 반영.
