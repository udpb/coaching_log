## 2. 본 감사의 추가 발견 (제품 차원)

### 🔴 P0 — 작성 중 데이터 유실
Draft 자동저장이 없다. transcript 붙여넣기 → AI 추출 → 검토 중 새로고침/탭 닫힘 = 전부 유실. localStorage 는 *저장된* 기록만 캐시 (index.html ~5064). **코치 신뢰를 한 번에 깎는 유형 — 퀵윈 1순위.**

### 🔴 P0 — 뒤로가기 = 사이트 이탈 (사용자 보고, 06-10)
해시 동기화 코드는 있었으나 기본이 `replaceState` 고 `pushHistory:true` 를 넘기는 호출자가 0곳 → 히스토리 엔트리가 안 쌓임. `#detail` 은 복원 자체를 포기. → **본 감사에서 수정 완료** (pushState 기본화 + `#detail/<id>` 복원, B-20260610-routing-history).

### 🟡 죽은 컬럼 11개 (69개 중 16%)
- `embedding` 계열 4개 (pgvector RAG — 코드 참조 0) → **"RAG 할 것인가" 제품 결정 필요 (ADR 후보)**
- `inferred_skills` 계열 3개 (추출·표시 모두 없음)
- `legacy_firestore_id` (이관 완료 잔재)
- `payment_info`·`task_summary`·`client`·`total_budget` (coach-finder 용인데 coaching_logs 에 잔존)

### 🟡 추출 버전·모델 미기록
API 가 `modelUsed`·`usage` 를 반환하지만 DB 미저장. 프롬프트 버전 없음 → 프롬프트 개선 시 과거 일지와 품질 비교 불가. `extraction_version`·`extraction_model` 컬럼 추가 권장 (싸고, 고도화 직전인 지금이 적기).

구조화 체크인 날짜·채널을 추출·저장까지 하면서 리마인더/할일 뷰가 없음. **"오늘 체크인할 팀" 뷰 하나로 커밋먼트 루프가 닫힘** — 데이터가 이미 있어 가장 싼 핵심 기능.
