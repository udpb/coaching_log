01

제품 경계 — 4앱 + 1공유 Supabase

언더독스는 4개 앱이 하나의 Supabase(coaches_directory 마스터)를 공유한다. coaching-log 는 그중 코치의 기록·추적을 담당한다.

앱페르소나책임
coaching-log
본 제품		 코치 코칭 일지(STT→AI)·성과 추적·코치 본인 정보 자가관리·BP/평가/코치배정(admin·pm)·수주(won) 트리거 발동 지점
coach-finderPM · admin코치 검색·AI 매칭·사업 기획·계약·예산
underdogs-hub전체4앱 진입
ud-opsadminDB · embedding · 인프라 · 제안서
수주 트리거가 여기서 발동된다. PM 이 사업기획(business_plan)을 수주(won) 처리하면 DB 트리거가 projects 를 만들고 accepted 코치를 project_members 로 흘려보낸다 → 그 순간부터 코치는 teams 홈에서 팀을 보고 세션 기록을 시작한다. (아래 ⑤·⑥ 참조)

안 하는 것 — 코치 검색/RAG 추천(→coach-finder) · 자가등록 신청 폼(→coach-finder /register) · RFP/커리큘럼/제안서(→ud-ops). ⚠️ 평가 작성은 coach-finder 와 같은 coach_evaluations 를 써서 중복(INTEGRATED_ARCHITECTURE Gap 3).

03

코치 핵심 여정 ★ — "세션 기록 → 성과 추적"

제품의 북극성 플로우. 코치가 말(또는 녹음)로 5분만 입력하면 AI 가 구조화 일지로 정리하고, 회차가 쌓이며 "이 팀이 나아지고 있나"가 한눈에 보인다.

🔐 로그인 (코치)

"STT 원문으로 초안 만들기" 패널
MODE A

≥ 50자. 외부 STT 도구 결과 paste (legacy)

▼

05

PM / admin 여정 — 사업기획 → 수주(트리거) → 평가

PM 이 사업을 기획·수주하면 코치가 자동으로 흘러 들어온다. coaching-log 와 coach-finder 를 잇는 핵심 연결고리.

🗒️ plans — 사업기획
business_plans 생성 (admin / pm) · 상태 draft → proposed → won / lost / cancelled
▼
코치 핀 — business_plan_coaches 후보 등록 (candidate → accepted)
🏆 수주 처리 status = 'won'
BP "수주 처리" 클릭
▼
DB 트리거 bp_on_won 발동 (자동)
⚙️ 자동 생성 (트리거)
① projects 생성  ·  ② BP 에 project_id 역기입  ·  ③ accepted 코치를 project_members 등록
▼
= 코치가 teams 홈에서 이 팀을 보게 됨 → 세션 기록 시작 (③ 여정으로 합류)
⭐ 평가 — BP 상세에서 5축 작성 (admin / pm)
종합 + 소통 + 전문성 + 성실함 (각 1–5) + 재계약의향(3-state) + 코멘트 → coach_evaluations
▼
🙈 코치 본인은 자기 평가 조회 불가 (RLS · 솔직성 보장)
